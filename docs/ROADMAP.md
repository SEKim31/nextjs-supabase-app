# 하루모아 (HaruMoa) 개발 로드맵

습관을 소중한 사람과 함께 인증하고 응원하며 지속 가능하게 만드는 커뮤니티형 루틴 앱

## 개요

하루모아(HaruMoa)는 혼자 하면 포기하기 쉬운 습관을 커플, 가족, 친구와 함께 만들어가는 서비스로 다음 기능을 제공합니다:

- **그룹 기반 습관 관리**: 최대 6명의 소중한 사람들과 함께 미션 수행
- **데일리 인증 시스템**: 사진/텍스트/체크리스트 방식의 유연한 인증
- **응원과 성취도 시각화**: "최고야!" 응원과 스트릭, 캘린더로 동기부여

---

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - API/비즈니스 로직 작업 시 "테스트 체크리스트" 섹션 필수 포함

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 테스트 통과 확인 후 다음 단계로 진행

4. **로드맵 업데이트**
   - 로드맵에서 완료된 작업을 완료로 표시

### 모바일 퍼스트 디자인 가이드라인

하루모아는 **모바일 앱 경험**을 데스크톱에서도 동일하게 제공하는 것을 목표로 합니다.

#### 레이아웃 원칙

- **최대 너비 제한**: 모든 콘텐츠는 `max-width: 430px`로 제한
- **중앙 정렬**: 데스크톱에서 콘텐츠가 화면 중앙에 배치
- **고정 높이 뷰포트**: `min-height: 100dvh`로 모바일 기기의 동적 뷰포트 대응
- **데스크톱 배경**: 선택적으로 그라데이션 또는 패턴 배경 적용 가능

#### 하단 탭 네비게이션

- **고정 위치**: `position: fixed; bottom: 0`로 하단 고정
- **5개 탭 구성**: 홈, 그룹, 인증(+), 통계, 설정
- **Safe Area 대응**: `padding-bottom: env(safe-area-inset-bottom)`로 노치/홈 인디케이터 영역 확보
- **활성 상태 표시**: 현재 페이지에 해당하는 탭 하이라이트

#### 스크롤 영역

- **콘텐츠 영역**: 하단 탭 높이(약 64px + safe area)를 고려한 `padding-bottom` 적용
- **스크롤 복원**: 페이지 이동 후 스크롤 위치 유지
- **Pull-to-refresh**: 향후 PWA에서 새로고침 제스처 지원 고려

#### 반응형 전략

```css
/* 기본 (모바일) */
.container {
  width: 100%;
  max-width: 430px;
  margin: 0 auto;
}

/* 데스크톱에서 앱 프레임 효과 */
@media (min-width: 768px) {
  .app-frame {
    box-shadow: 0 0 40px rgba(0, 0, 0, 0.1);
    border-radius: 24px; /* 선택적 */
  }
}
```
